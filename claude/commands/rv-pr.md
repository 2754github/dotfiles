# rv-pr

ultrathink

$1 を下記の観点でRVし、

- セキュリティ（脆弱性）
- 不具合
- パフォーマンス（性能）
- 人間にとっての可読性
- 既存コードとの一貫性
- SOLID原則
- DRY原則
- KISS原則
- YAGNI原則

下記のフォーマットで出力する。

```md
### 良い点

- <良い設計>
- <良い実装>

### 改善点

- <セキュリティや不具合などの重大なもの>
- <すぐに改善できるもの>
- <改善に時間がかかるもの>
```

---

下記でPR概要を確認できる。

```sh
gh pr view <PR番号>
```

下記でコード差分を確認できる。

```sh
gh pr diff <PR番号>
```
